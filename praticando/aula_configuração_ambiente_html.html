Os detalhes a respeito das linhas de comando e codigo podem ser vistas no site:

CMD:
> mkdir 'name' --- Cria um repositorio com o name designado
> dir --- lista os arquivos do repositorio atual
> cd 'name' --- navega para dentro do repositorio name.
> cd.. --- navega para um nivel anterior do diretorio atual.

GIT:
> git clone 'url' --- utilizado para clonar os arquivos localizados no diretorio da url informada. A Url pode ser obtida no site do repositorio acessando a conta do GitHub pelo browser.
> git status --- utilizado para verificar o status (mudanças) existentes no diretorio atual.
> git chekout 'branch' --- utilzado para mudar da branch atual para a cranch especificada no comando.
> git add . --- utilizado para adicionar todos as arquivos alterados para uma modificação.
> git commit -m 'menssagem' --- utilizado para escrever a modificação e adicionar a menssagem incluida.
> git push --- utilizado para empurrar todos os commits para a branch que está em utilização.

BROSER SYNC:
> Comando: browser-sync start -- server --directory --files "*"
Comando utilizado para iniciar a sincronização do diretorio em, utilização com o navegador.
-- O Parametro start inicia o srviço
-- O paramento --server inicia um servidor HTTP local, por padrão na porta 3000
-- O parametro directory fornece uma lista dos diretorios da pasta atual
-- O parametro files é utilizado para setar os arquivos que terão a sincronização ativada. Neste caso o parametro "*" é utilizados para setar todos os arquivos do diretorio.
>Com a sincronização ativada, as modificações aplicadas nos arquivos .html podem ser vistas em tempo real apenas executando o comando de salvar o arquivo (Ctrl+s). Isso é util para não ter que atualizar a pagina constantemente.
